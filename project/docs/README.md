# Project Documentation

Project-specific operational documentation for the payroll lakehouse platform.

## Structure

```
docs/
├── README.md                  # This file
├── architecture.md            # Detailed architecture (layer-by-layer)
├── data_model.md              # ERDs, schema documentation
├── dataops.md                 # CI/CD pipeline, DQ gates
├── finops.md                  # Cost optimization strategies
├── privacy.md                 # PII governance, access control
├── runbook.md                 # Operational runbook (incidents)
├── deployment.md              # Deployment procedures
└── postmortem.md              # Lessons learned, retrospective
```

## Documentation Philosophy

- **Living docs**: Update as system evolves
- **Actionable**: Every doc has clear procedures
- **Visual**: Diagrams and examples included
- **Searchable**: Good headings and organization
